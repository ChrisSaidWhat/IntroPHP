Git Terms

Citation: Traversy Media

Version Control Software

Version control software is a tool that track changes made to code over its history and links changes to individual contributors. The code can be shared to many different machines, worked on separately, and then merged again centrally without the chance of conflicting changes breaking the code. This allows for teams to work faster, in a decentralized manner, with greater accuracy. (Geeks For Geeks, Traversy Media)

Add

The Git add command sends the file and its changes to a staging area where a snapshot is taken. This snapshot can be updated many times and will not leave local machine until it it is committed. This is the first step for tracking a file. Git does not track it until you add it. (Git Documentation, Traversy Media)

Commit

The Git commit command takes the snapshot and adds it to the repository on the local machine. Until the snapshot is committed it remains in the staging area and is not tracked nor is it in the repository. (Git Documentation, Traversy Media)

Push

The Git push command updates "remote refs along with associated objects," according to the Git docs. However, the main use of the push command is to send the contents and changes of the local repository to the remote repository on GitHub or one of the similar tools. It takes when we have added and committed and then puts them into the remote repository where it can be kept safe, updated, shared, and contributed to by others without breaking the original code or any added thereafter. (Git Documentation, Traversy Media)

Pull

The Git pull command fetches "from and integrates with another repository or local branch," according to the Git docs. Basically, the pull command can be thought of as the opposite of the push command. This command takes the contents of the remote repository and puts them on the local machine in a local repository. It also behind the scenes is performing merges and fetches which is beyond the scope of the current assignment and conversation.When the dev is done with their changes, they will again perform a push command to send the new code to the remote repository. (Git Documentation, Traversy Media)

Clone

The Git clone command duplicates a remote repository and installs it on the local machine in a new directory. This allows the dev to work on their own copy of code without effecting the original codebase. This also enables collaboration between devs from around the world without any physical interaction needed. As with the pull command, the clone command involves branches and merging which is beyond the scope of the current assignment and conversation but it is valuable to note. (Git Documentation, Traversy Media)
